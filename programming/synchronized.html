Java의 Synchronized에 Deadlock 발생에 대해 알아보자.

얼마 전에 면접을 봤었다. Synchronized 키워드에 대해 질문을 받았었는데, 어찌 저찌 하다 보니 Deadlock에 대해서도 질문이 이어졌다. 대강 기억해 보자면, 'Synchronized가 붙은 객체를 사용하면 Deadlock이 걸리나?' 였는데, 'Synchronized가 붙은 객체를 사용해도 Deadlock이 걸릴 수 있다!'라고 대답 했지만 내가 질문을 잘못 이해 했던 건지 아니면 대답을 잘못 했는지...(아니면 면접관이 잘못 알고 있었던게 아닐까 의심이...) 꼬리질문을 받다 결국 잘못 알고 있는 것 같다고 대답하고 마무리 했었다.(결국 떨어짐)

그래서! Synchronized에 대해 살짝 알아보고, Synchronized 키워드가 붙은(일반 메서드, static 메서드, 코드블럭 어디든) 객체를 사용할 때 Deadlock이 걸리는지 에 대해 테스트를 해보려 한다.

사용한 클래스는 아래와 같다.

import java.util.ArrayList;
import java.util.List;

public class TreeNode {
    TreeNode parent = null;
    List children = new ArrayList();

    public synchronized void addChild(TreeNode child) {
        System.out.println(Thread.currentThread() + " TreeNode.addChild");
        try {
            Thread.sleep(2000);
        } catch (Exception e) {
            e.printStackTrace();
        }
        if (!this.children.contains(child)) {
            this.children.add(child);
            child.setParentOnly(this);
        }
    }

    public synchronized void setParent(TreeNode parent) {
        System.out.println(Thread.currentThread() + " TreeNode.setParent");
        this.parent = parent;
        parent.addChildOnly(this);
    }

    public synchronized void addChildOnly(TreeNode child) {
        System.out.println(Thread.currentThread() + " TreeNode.addChildOnly");
        if (!this.children.contains(child)) {
            this.children.add(child);
        }
    }

    public synchronized void setParentOnly(TreeNode parent) {
        System.out.println(Thread.currentThread() + " TreeNode.setParentOnly");
        this.parent = parent;
    }

    public void synchronizedBlockMethod() {
        synchronized (this) {
            try {
                for (int i = 0; i < 3; i++) {
                    System.out.println(Thread.currentThread() + " :: synchronizedBlockMethod");
                }
            } catch (Exception e) {
                e.printStackTrace();
            }
        }
    }

    public synchronized void synchronizedMethod() {
        try {
            for (int i = 0; i < 3; i++) {
                System.out.println(Thread.currentThread() + " :: synchronizedMethod");
            }
        } catch (Exception e) {
            e.printStackTrace();
        }
    }

    public void method() {
        for (int i = 0; i < 3; i++) {
            System.out.println(Thread.currentThread() + " :: method");
        }
    }
}

테스트 1 - Synchronized 의 lock 범위

테스트 목적

  • Synchronized 키워드 가 붙은 위치에 따라 Lock이 걸리는 범위를 알아 본다.

설명

  • Thread1 생성 - Synchronized 블럭 있는 메서드 실행 - a
  • Thread2 생성 - Synchronized 메서드 실행 - b
  • Thread3 생성 - 일반 메서드 실행 - c

예상 결과

  • a, b 중 하나 실행
  • c와 a, b 중 하나 동시 실행

테스트 코드

@Test
public void synchronizedTest() throws InterruptedException {
    TreeNode parentTreeNode = new TreeNode();

    Thread thread1 = new Thread(() -> {
        System.out.println(Thread.currentThread());
        parentTreeNode.synchronizedBlockMethod();
    });

    Thread thread2 = new Thread(() -> {
        System.out.println(Thread.currentThread());
        parentTreeNode.synchronizedMethod();
    });

    Thread thread3 = new Thread(() -> {
        System.out.println(Thread.currentThread());
        parentTreeNode.method();
    });

    thread1.start();
    thread2.start();
    thread3.start();
    thread1.join();
    thread2.join();
    thread3.join();
}

Thread[Thread-5,5,main] :: method                  - c

Thread[Thread-4,5,main] :: synchronizedMethod      - b
Thread[Thread-4,5,main] :: synchronizedMethod      - b

테스트 2 - Deadlock이 진짜 걸리는가?

테스트 목적

  • 메서드에 synchronized 키워드가 붙은 메서드 호출 시 데드락 발생 테스트

테스트 순서

  1. Thread 1에서 parentTreeNode 의 addChild 메소드 호출
  2. Thread 2에서 childTreeNode 의 setParent 메소드 호출
  3. Thread 1에서 child.setParentOnly(this) 호출
  4. Thread 1에서 parent.addChildOnly(this) 호출

예상 시나리오

- Thread 1 Thread 2
1 parentTreeNode 의 addChild 메소드 호출
    -- parentTreeNode 객체 lock 걸림
2 sleep 2 sec
3 sleep 1 sec
4 childTreeNode 의 setParent 메소드 호출
    - childTreeNode 객체 lock 걸림
5 parentTreeNode 의 lock 이 풀리지 않은 상태로 child.setParentOnly(this) 호출
    - parentTreeNode lock 인 상태 에서 childTreeNode 에 lock 을 걸기 위해 대기
6 childTreeNode 의 lock 이 풀리지 않은 상태로 parent.addChildOnly(this) 호출
    - childTreeNode lock 인 상태 에서 parentTreeNode 에 lock 을 걸기 위해 대기
7 Deadlock is occurred
